# DailyFlora Member UI Direction 01

最后更新：2026-07-13

## 本稿解决什么

本方向稿不是 Member 的最终实现，也不是只讨论颜色和字体的 moodboard。它用于确认免费用户、付费用户、收藏、生成资产、平台采用、积分和施工状态在同一套 UI 中如何分层。

当前默认状态是“免费用户 + 有收藏”。顶部原型工具允许切换免费/付费和有收藏/空收藏，用于检查四种关键状态。

## 核心判断

1. Member 的第一层始终是“我的收藏”。付费不会改变这一层的优先级。
2. 免费用户在收藏之后看到功能说明、真实案例、权益解释和积分/施工状态。
3. 付费用户在收藏之后进入上传与生成、已生成花花管理、积分流水和通讯工作区。
4. Member 不重复展示首页的“今日花花”，也不存在“我的花束日历”。
5. 项目正在开发、MVP 占位和施工状态继续保留，但必须使用正式 UI 组件呈现，不能写成散落的开发备注。

## 页面状态矩阵

| 用户状态 | 收藏状态 | 收藏之后的主要内容 | 首要行动 |
| --- | --- | --- | --- |
| 免费 | 有收藏 | 案例种草、功能说明、积分解释、付费能力预览 | 进入收藏日期、了解生成方式 |
| 免费 | 空收藏 | 空收藏引导，然后继续展示免费转付费内容 | 回首页收藏第一束花 |
| 付费 | 有收藏 | 上传与生成、我的已生成花花、积分流水 | 生成新花、管理已有花花 |
| 付费 | 空收藏 | 空收藏引导，然后进入付费工作区 | 收藏花花或直接生成 |

## 我的收藏

收藏是免费用户最高频入口，首屏采用一张主收藏加两张辅助收藏的结构。

每张收藏卡包含：

- 花束真实画面
- 日期或特别花束标记
- 花名
- 进入对应日期/花束
- 取消收藏

取消收藏属于可恢复操作。正式实现建议出现短暂撤销提示；本方向稿只模拟移除和空状态切换。

空收藏状态不使用虚构收藏填充，只说明收藏价值并返回首页。

## 免费用户转化顺序

1. 先看到自己已经喜欢的成果。
2. 再理解“上传参考 + 审美偏好 + 生成私人花花”的能力。
3. 通过一个完整案例理解私人 URL、平台采用、公开日期入口和积分奖励。
4. 阅读四项付费能力，而不是直接进入不可用表单。
5. 最后解释积分和充值建设状态。

积分区不虚构价格套餐。当前只表达积分用途、采用奖励和充值入口的施工状态。

## 付费后的已生成花花管理

### 上传参考图到生成记录

正式流程不包含“用户主动导出”。用户确认上传后，系统自动将参考图处理为一张主参考 WebP 和一张 320px 小缩略 WebP；生成成功后自动追加到“我的已生成花花”。用户列表和超管后台用缩略图帮助回忆当时上传了什么，按权限可查看主参考图。

两张图片存对象存储，一条参数记录存数据库。识别只匹配系统公共花材库，不为用户新建花材，用户记录中也不保存花材列表。完整字段和权限边界见 [`user-reference-generation-contract.md`](./user-reference-generation-contract.md)。

### 未被平台采用

- 私人名称可修改。
- 可进入仅本人可视 URL。
- 可删除。
- 显示生成日期和积分消耗。

### 已被平台采用

- 显示平台公开名称和采用日期。
- 显示进入对应每日花花的快捷入口。
- 平台公开名称锁定，用户修改名称只改变私人备注名。
- 保留仅本人可视 URL。
- 显示采用奖励积分。
- 可针对具体花花发起改名建议工单或站内信。

用户删除已采用花花时，正式产品必须区分“从我的列表移除”与“删除平台公开记录”。方向稿暂不定义后端删除权限。

## 积分流水

每条流水至少包含：

- 日期
- 事项
- 关联花花
- 增加或扣除数额
- 变动后余额
- 若由平台采用产生，关联采用日期

余额可以在页头快速查看，但详细流水属于付费工作区。

## 工单与站内信

工单必须从具体花花发起，并自动关联：

- 花花 ID
- 用户私人备注名
- 平台公开名称
- 采用日期
- 工单类型
- 用户说明

本方向稿只在已采用花花卡片中保留入口，下一轮再深化完整通讯线程。

## 施工状态组件

统一使用以下状态词：

- MVP 可体验
- 仅本地保存
- 功能建设中
- 等待平台接入
- 暂未开放
- 模拟数据

施工卡片应说明“功能价值、当前可用部分、尚未开放部分”，不能只显示一个灰色禁用按钮。

## 视觉系统

- 主背景：`#050705`
- 内容背景：`#0B100C` / `#101710`
- 主文字：`#F5F0E6`
- 植物强调：`#C7DC8D`
- 积分/施工强调：`#EAD36F`
- 采用成功：`#9FD47C`
- 花名与情绪标题：Iowan Old Style / Baskerville / Songti SC
- 功能文字：Inter / 系统无衬线

卡片只在承载花束画面、付费能力预览或明确状态时出现。普通叙事内容使用分割线和留白组织，避免把 Member 做成通用 SaaS 仪表盘。

## 响应式规则

- `>1080px`：一张主收藏 + 两张辅助收藏；案例左右分栏；付费工作区双栏。
- `721–1080px`：主收藏单独一行，两张辅助收藏并排；其他双栏改为单栏。
- `≤720px`：收藏全部单列；隐藏页头文字导航；按钮可换行；已生成花花操作移动到卡片底部。

手机端仍然先完整展示收藏，不把收藏压缩成横向难操作的小卡片。

## 本稿暂不实现

- 真实登录与用户权限
- 图片上传与识别
- 真实积分扣除和充值
- 数据库持久化
- 工单消息发送
- 平台采用后台

这些能力可以保留入口、状态和真实字段，但不能在方向稿中伪装为已经完成。
