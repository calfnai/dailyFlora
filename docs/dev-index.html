路由图

正式数据与存储分布

数据域 低成本 MVP 做法 未来数据库职责
users 先用假数据或静态 JSON 展示资料页。 用户资料、联系方式、头像、状态；正式权限以后再接认证。
bouquets 固定样例先写在页面数据里。 每日展示、精选状态、来源用户、日期、主题、seed、公开状态。
generations member-test 用 mock 列表表达流程。 每次生成一条追加式记录：所属用户、主参考图和缩略图的存储定位、识别结果、颜色、构图、seed、Three.js 参数、系统版本和结果状态；不保存用户花材列表。
uploads 只展示上传卡片，不实际上传。 主参考 WebP 与 320px 缩略 WebP 进入私有对象存储，不另建重复的用户上传数据库记录；两个文件通过同一个生成记录 ID 关联。
credits 只展示积分流水样例，不接支付。 发放、消耗、获奖返还、人工调整；余额可缓存，但流水必须保留。
admin_audit_logs admin-users 先展示操作日志样式。 记录操作员、时间、IP、原因、变更前后值，尤其用于积分和用户资料。
