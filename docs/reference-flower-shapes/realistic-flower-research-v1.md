# DailyFlora 偏写实花型校准研究 v1

本记录服务于 `docs/realistic-flower-lab.html` 的程序化几何校准。目标是先保住物种识别所依赖的花序、单花结构和开放顺序，再处理颜色与审美。模型仍是低面数、程序化的偏写实表达，不是植物学扫描模型。

花型与叶片共用的参考选择、隔离验证、双重验收和冻结规则见：[偏写实植物器官研究方法 v1](./realistic-organ-research-method-v1.md)。本记录中的逐物种规则继续负责具体花序和单花结构；共用方法只约束研究与验收流程，不把不同物种合并回通用几何模板。

## 1. 绣球 / Hydrangea

- 目标物种或园艺类型：`Hydrangea macrophylla` mophead cultivar group（大叶绣球球形花序园艺类型）。
- 花序类型：顶生大型聚伞／伞房状复合花序；mophead 园艺型远看呈半球至近球形。
- 单花结构：视觉主体是 4–5 裂、显著扩大的不育装饰花萼片；内部仍混有较小的可育花、未开花和不同成熟度。
- 开放顺序：同一花球内不同小簇不同步，外层装饰花、内部小花和花蕾并存。
- 远看轮廓：饱满但不是数学球体，局部有凹凸、遮挡和轻微缺口。
- 当前代码原问题：104 朵相同四萼片花按规则球壳铺设；花朵大小和开放状态几乎一致；只有少量中心辐条，缺少小簇层级。
- 本次生成规则：主茎连接中心汇合点；一级短枝连接多个不规则小簇；二级短枝连接装饰花、半开花和花蕾；花球使用偏心、扰动半径与局部空缺，避免完美球壳。
- 仍未解决的问题：未模拟具体品种的萼片锯齿、脉纹、可育花雄蕊和真实花序发育时间差。
- 参考来源：[NC State Extension — Hydrangea macrophylla mophead](https://plants.ces.ncsu.edu/plants/hydrangea-macrophylla/common-name/mophead-hydrangea/)

## 2. 满天星 / Baby's Breath

- 目标物种或园艺类型：`Gypsophila paniculata`，以切花常见单瓣型为几何基准。
- 花序类型：高度分枝的圆锥花序／聚伞状分枝系统。
- 单花结构：枝端小型白花，通常 5 瓣；花蕾近圆，细短花梗连接末级分枝。
- 开放顺序：枝端同一小群内花蕾、半开和盛开混合，不是每条枝固定相同花数。
- 远看轮廓：枝条细、分叉多、末端花密，形成不规则空气花雾。
- 当前代码原问题：12 条等距一级枝，每条固定 6 朵；分枝长度、角度和层级过于一致，像天线阵列。
- 本次生成规则：主轴上随机布置一级枝；一级枝再生成 2–4 条不等长二级枝；末端生成 1–4 条短花梗；花只出现在真实枝端并混合三种开放阶段。
- 仍未解决的问题：未区分单瓣与重瓣切花品种，也未模拟极细叶片和花瓣边缘细节。
- 参考来源：[NC State Extension — Gypsophila paniculata](https://plants.ces.ncsu.edu/plants/gypsophila-paniculata/common-name/babys-breath/)

## 3. 蛇鞭菊 / Liatris

- 目标物种或园艺类型：`Liatris spicata`。
- 花序类型：多个近无柄头状花序密集排列成穗状花序。
- 单花结构：每个花头由约 4–10 枚管状盘花组成，没有普通舌状“五瓣花”；花柱明显外伸。
- 开放顺序：典型自顶部向下开放，下部保留未开花头。
- 远看轮廓：细长、紧贴轴线的紫色瓶刷，上部开放区更绒、更亮。
- 当前代码原问题：把每个位置做成普通五瓣花，且通过明显花梗离开主轴。
- 本次生成规则：头状花序直接贴合主轴；每头由多枚细管状小花和外伸花柱组成；顶部盛开、中部半开、下部花蕾；低面数圆柱与细线使用实例化。
- 仍未解决的问题：总苞片只做轮廓提示，未逐片模拟；花柱分叉和花药细节仍被简化。
- 参考来源：[NC State Extension — Liatris spicata](https://plants.ces.ncsu.edu/plants/liatris-spicata/common-name/sessile-headed-blazing-star/)

## 4. 蕾丝花 / Lace Flower

- 目标物种或园艺类型：`Ammi majus`。
- 花序类型：大型复伞形花序；一级伞梗数量多且不完全等长，末端为含多朵小花的小伞形花序。
- 单花结构：约 1 mm 白色五瓣微小花，末端短花梗长度不一；小伞形花序下有窄苞片线索。
- 开放顺序：同一复伞形花序内小伞形花簇可处于花蕾、半开和盛开混合状态。
- 远看轮廓：接近平面或轻微拱起，边缘松散，不是完美圆盘或规则烟花。
- 当前代码原问题：20 根一级伞梗按黄金角规则排布，每个小伞固定 6 花，缺少总苞片、次级苞片和密度差。
- 本次生成规则：中心节点发出不等长一级伞梗；每个末端小伞生成 7–12 根不等长二级花梗；中心伞梗略短，整体轻拱；中心总苞片与小伞苞片用细线提示。
- 仍未解决的问题：未达到真实个体 15–50 条一级伞梗和每小伞约 18–30 花的完整数量，以控制实时渲染成本。
- 参考来源：[Kew Plants of the World Online — Ammi majus](https://powo.science.kew.org/taxon/urn%3Alsid%3Aipni.org%3Anames%3A837486-1/general-information)

## 5. 风信子 / Hyacinth

- 目标物种或园艺类型：`Hyacinthus orientalis` 园艺型。
- 花序类型：短而密、圆柱状的多花总状花序。
- 单花结构：花被基部合生成管状至钟形花冠，顶端 6 裂，裂片向外展开并反卷；每朵以很短花梗接轴。
- 开放顺序：下部和中部多为盛开花，上部混有半开花与花蕾。
- 远看轮廓：短、粗、密集的圆柱花序，不是细长穗或六片分离花瓣毛刷。
- 当前代码原问题：六片独立通用花瓣围绕球体，没有合生花冠筒，也没有清楚的反卷裂片。
- 本次生成规则：每朵由一个开口钟形花冠筒与 6 个反卷裂片组成；花冠筒、短花梗和主轴连续；顶部使用封闭花蕾与半开筒形花冠。
- 仍未解决的问题：未模拟花冠厚蜡质感、香气结构、花药与不同园艺品种的重瓣化。
- 参考来源：[NC State Extension — Hyacinthus orientalis](https://plants.ces.ncsu.edu/plants/hyacinthus-orientalis/)

## 6. 狐尾百合 / Foxtail Lily

- 目标物种或园艺类型：`Eremurus robustus` 与常见 `Eremurus` 切花园艺型。
- 花序类型：高大、挺拔、密集的圆柱状总状花序。
- 单花结构：6 枚星形花被片；通常 6 枚明显外伸雄蕊与花药；花通过花梗接入无叶主轴。
- 开放顺序：下部盛开，中部半开，顶部密集花蕾；花梗从下向上逐渐缩短。
- 远看轮廓：长圆柱或狐尾状，轴线基本直立，仅有轻微自然偏摆。
- 当前代码原问题：仍是通用六瓣穗花模板；缺少外伸雄蕊、花药、花梗梯度和阶段分区。
- 本次生成规则：独立生成花梗长度梯度、6 花被片、6 雄蕊和花药；下中上三段分别映射盛开、半开、花蕾；主轴使用小幅曲线。
- 仍未解决的问题：真实花序可达数百朵，本模型为性能只保留约四十个可辨识单花；未模拟花被中脉。
- 参考来源：[NC State Extension — Eremurus](https://plants.ces.ncsu.edu/plants/eremurus/common-name/foxtail-lily/)

## 7. 金鱼草 / Snapdragon

- 目标物种或园艺类型：`Antirrhinum majus` 高秆切花型。
- 花序类型：顶生总状花序，下部花较大，上部逐渐过渡到花蕾。
- 单花结构：合生花冠筒；花冠左右对称、上下二唇；上唇 2 裂，下唇 3 裂；下唇中部隆起并封闭或半封闭喉部；基部花萼 5 裂。
- 开放顺序：下部盛开，中部半开，顶部花蕾。
- 远看轮廓：单花侧向突出，能从侧面读出花冠筒与闭喉两唇，而不是普通花瓣球。
- 当前代码原问题：仅用三片通用花瓣围绕椭球，缺少合生花冠筒、上下两唇、隆起喉部和五裂花萼。
- 本次生成规则：每朵使用独立筒状花冠、2 个上唇裂片、3 个下唇裂片、中央喉部隆起和 5 个花萼裂片；短花梗连接主轴；三段开放阶段分别改变筒长、唇瓣展开度和花蕾比例。
- 仍未解决的问题：闭喉内部的蜜腺、细毛和雄蕊隐藏关系仍为简化；花冠褶皱未做扫描级细节。
- 参考来源：[NC State Extension — Antirrhinum majus](https://plants.ces.ncsu.edu/plants/antirrhinum-majus/common-name/common-snapdragon/)

## 8. 飞燕草 / Delphinium

- 目标物种或园艺类型：`Delphinium elatum` hybrid group，并用 `Delphinium` 属的稳定花部特征校准。
- 花序类型：纤细花轴上的总状至圆锥状花序，单花错落并通过短花梗连接。
- 单花结构：5 枚最醒目的花瓣状萼片；后方一枚萼片连续延长成花距；内部另有 2–5 枚较小真花瓣；中心多雄蕊且明暗对比强；花左右不完全对称。
- 开放顺序：下部盛开，中部半开，顶部花蕾。
- 远看轮廓：纤细、开放、有花间距；每朵花背后的花距应与后萼片连续。
- 当前代码原问题：主体是普通五瓣花，内部真花瓣和中心层次不足；花距像额外插上的尖片。
- 本次生成规则：后萼片与锥形花距共享同一连接基部；其余四萼片形成不对称外轮廓；内部小花瓣独立实例化并用浅色中心对比；短花梗连接花轴，顶部使用花蕾。
- 仍未解决的问题：真花瓣的二裂形态、无限多数雄蕊和不同杂交品种的重瓣程度仍被简化。
- 参考来源：[Kew Plants of the World Online — Delphinium](https://powo.science.kew.org/taxon/urn%3Alsid%3Aipni.org%3Anames%3A328251-2/general-information)、[NC State Extension — Delphinium exaltatum](https://plants.ces.ncsu.edu/plants/delphinium-exaltatum/)

## 性能与连接约束

- 重复花瓣、花冠筒、花蕾、花药和花柱优先合并为各花型内部的 `InstancedMesh`。
- 主轴可使用单条 `TubeGeometry`；大量短枝、花梗和雄蕊用单位圆柱实例化并按两端点变换。
- 每一朵花的位置必须由轴点、分枝节点或花梗端点推导，不允许直接在空间撒点。
- 所有开放阶段仍共享同一物种的专用结构，只改变展开度和器官可见程度，不退回通用花瓣模板。
