# DailyFlora 偏写实植物器官研究方法 v1

日期：2026-07-22

适用范围：叶片、单花、花序、花梗、分枝与其他需要从真实形态转换为程序几何的植物器官。

## 核心结论

真实参考负责边界，程序只负责受限转换；每次只修改一个层级，工程检查与形态验收分开，达到方向验证后及时冻结。

## 1. 先确定研究对象

- 先锁定物种、园艺类型或明确的几何参照，避免用含混俗名直接生成形态。
- 把术语拆开记录。例如掌状脉、单叶、五裂轮廓是三个不同维度，不能把 `palmate` 直接翻译成五根手指。
- 先列出决定识别度的结构：器官拓扑、连接关系、远看轮廓、开放或生长阶段，再处理颜色和装饰细节。

## 2. 隔离验证层级

形态任务按以下顺序独立验收：

1. 真实参考和目标结构。
2. 二维边界或器官拓扑。
3. 三维截面、曲面与局部姿态。
4. 器官连接和生长层级。
5. 单成员原型。
6. 完整植物成员映射。
7. 主花束构图和性能。

某一层未通过时，不用颜色、材质、随机扰动或下一层细节掩盖它。

## 3. 参考图到程序几何

- 优先使用植物园、大学数据库、Kew、Wikimedia Commons 或可追溯植物学图像。
- 轮廓以参考图 overlay 和人工结构地标为准，不让自由曲线或自动指标重新设计植物。
- 地标只描述主要器官、裂片、叶窦、肩部和连接点；小锯齿等次级细节不得主导大结构。
- 二维 baseline 冻结后，三维阶段只能增加厚度、凹凸、局部姿态和连续连接，不能暗中改写前视边界。

## 4. Codex 的适用边界

Codex 适合执行：

- 把已确认的参考地标转换为受约束曲线或网格。
- 建立独立调试页、固定视图、overlay、wireframe 和 section。
- 检查自交、翻面、悬浮组件、连接断裂、构建与性能。
- 在已锁定结构内实现确定性、可重复的轻微变化。

Codex 不应自行决定：

- 哪个自由生成轮廓更像真实植物。
- 用更多参数替代缺失的真实参考。
- 用统一模板和缩放代替不同器官拓扑。
- 用工程指标或编译通过宣称形态正确。

## 5. 双重验收

### 工程验收

- TypeScript、构建和 diff 检查通过。
- 网格无自交、翻面、破面和悬浮连接。
- 固定 seed、花朵数据和主构图在非目标任务中不漂移。
- 调试页和资源在真实浏览器可打开。

### 形态验收

- 与真实参考 overlay 对照。
- 检查 front、back、side、top、perspective、wireframe 和 section。
- 检查器官正反面、截面、连接点、遮挡与生长层级。
- 由人工视觉决定是否通过；自动 displacement、面积和比例只作辅助。

## 6. 冻结机制

原型状态必须明确区分：

- `exploration-only`
- `direction-validated-not-production-ready`
- `3d-mini-lab-frozen-not-system-integrated`
- `member-mapped-awaiting-bouquet-review`
- `production-ready`

达到本轮目标后记录未解决问题并冻结。冻结不是宣称完成，而是阻止原型在局部参数循环中继续退化。

## 7. 对写实花型研究的直接应用

- 花序先验证轴、分枝和开放顺序，单花再验证器官数量、合生关系与不对称性。
- 每朵花必须从主轴、分枝节点或花梗端点推导，不能在空间撒点。
- 花蕾、半开和盛开属于同一物种结构的阶段变化，不能回退为三套通用形状。
- 远看轮廓、单花器官和连接拓扑分别验收，避免“整体好看”掩盖关键器官错误。
- 每个物种采用独立结构规则；共享工具仅处理管线、连接、实例化和性能。

## 8. 本轮叶片实验带来的限制

- Strap 只完成方向验证，根部叶鞘连续拓扑仍未解决。
- Palmate 只完成独立 Major Structure Envelope 与一次受控 3D 曲面实验。
- 两者均未绑定真实植物成员，也未接入主花束。
- 后续若重新开启叶片研究，应从已冻结状态继续，不能重新使用通用叶片模板。
